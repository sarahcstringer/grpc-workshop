What is gRPC?

tl;dr: A framework for connecting services

gRPC Overview

  • g(meaning varies) Remote Procedure Call
  • Runs over HTTP/2
  • Uses Protocol Buffers for data serialization
  • Supports request/response and streaming
  • Allows code generation of server and client stubs in 11 languages

Protocol Buffers
Sample .proto File 

						syntax = "proto3";

						package images;

						message Empty {}

						message User {
						  string username = 1;
						  string email = 2;
						}

						message Image {
						  int32 id = 1;
						  User user = 2;
						  int32 width = 3;
						  int32 height = 4;
						  bytes image_data = 5;
						}

						message LikeImageRequest {
						  User user = 1;
						  int32 image_id = 2;
						}

						service ImageLiker {
						  rpc AddUser(User) returns (Empty) {}
						  rpc AddImage(Image) returns (Empty) {}
						  rpc ViewImages(Empty) returns (stream Image) {}
						  rpc LikeImage(LikeImageRequest) returns (Empty) {}
						}
Creating a gRPC Service

Creating a gRPC Service

  1. Define the service using Protocol Buffers
  2. Compile protobufs and generate server and client code stubs
    • Example for python: 
      python3 -m grpc_tools.protoc -I=. --python-out=. --grpc_python_out=. service.proto
    • Generates the files:
      • service_pb.py (defines request and response classes in Python)
      • service_pb_grpc.py (defines server and client stubs in Python)
  3. Implement server and client logic using the generated stubs
